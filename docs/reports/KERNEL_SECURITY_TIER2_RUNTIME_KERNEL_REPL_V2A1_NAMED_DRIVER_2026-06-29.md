## What This Adds

v1-repl already proved the four ops (`slide`/`peek`/`poke`/`call`) live, but each was
driven by hand against raw runtime addresses. v2a1 closes the gap opened by the v2a0
kallsyms fix (commit `ab480fea`): symbols now resolve **by name** from the corrected
`a90_stock_kallsyms_extract.py` System.map, and the driver computes

    runtime_vaddr(symbol) = link_vaddr(symbol) + slide

where `slide` comes live from op 0 (`adr x1,.` self-PC leak). The driver then drives
named `peek`/`call` over the existing image. No reflash is needed per experiment; the
only flash in the live unit is the one-time v1-repl image (already validated) plus the
V2321 rollback.

### Transport

Both the command-buffer write and the dmesg read go through the native-init `run`
builtin (`["run", busybox, "sh", "-c", ...]`), whose child stdout is captured into the
A90P1 console block. The write uses `printf` octal escapes into
`/sys/class/kgsl/kgsl-3d0/force_no_nap`; each op printk's one `A90R%llx` line, read via
`dmesg | tail`.

### Command-buffer layout (matches the v1-repl builder)

    +0x00 u64 magic (0xA90C0DE5DEADBEEF)   +0x08 u8 op
    +0x10 arg0 (peek/poke addr | call target)
    +0x18 arg1 (peek len | poke val | call x0)
    +0x20 arg2 (poke width | call x1)      +0x28..+0x50 call x2..x7

### Live transport findings (pinned for later units)

- The USB-ACM bridge is *not* the kernel console UART, so kernel `printk` is only readable
  via the log ring (`dmesg`/`/dev/kmsg`), not inline on the serial stream.
- busybox `dmesg` on this device is **read-and-clear** (consuming): each read returns the
  ring and clears it. The op driver therefore writes the command buffer and reads the
  `A90R` line **in one `run` shell**, bounded with `tail -n N`, because the native-init
  `run` console capture truncates a raw unbounded `dmesg` and loses the trailing line.
- **`kallsyms_lookup_name` is unsafe to call this way**: a live `call kallsyms_lookup_name`
  with an existing kernel "printk" string pointer reset the device (recoverable ‚Äî it reboots
  into the v1-repl boot partition). The call proof was switched to the v1-repl-proven safe
  `printk` target. A symbol-name ‚Üí address lookup that needs `kallsyms_lookup_name` is
  deferred until its call context is understood.

## Safety

- No new boot image; the only flashes are the already-validated v1-repl image and the
  V2321 rollback (boot-partition only, checked helper, readback verify).
- Driver issues `peek` and `call` only; it never issues `poke` in the v2a1 selftest.
- `call` targets are real JOPP function entries verified against the static image.
- No forbidden-partition writes, no RKP bypass, no protected-memory write, no RWX.
- v2a2 (kmalloc `poke` round-trip via `__kmalloc`/`kfree`) remains a separate unit.
